1. Algorithme

Un algorithme est une suite finie d’instructions précises permettant de résoudre un problème. C'est la "recette de cuisine" du programmeur avant même de toucher au code.

Les structures de base

1. La Séquence

Exécution linéaire d’instructions.

Début
   Lire A, B
   Somme ← A + B
   Afficher Somme
Fin

2. La Condition (Si…Alors)

Début
   Lire N
   Si N mod 2 = 0 Alors
      Afficher "Pair"
   Sinon
      Afficher "Impair"
   Fin Si
Fin

3. La Boucle (Répétition)

Début
   Pour i de 1 à 10 Faire
      Afficher i
   Fin Pour
Fin

2. Langage C

Créé en 1972, le C est un langage bas niveau puissant, utilisé pour les systèmes d'exploitation et l'embarqué. Il permet un contrôle total de la mémoire.

Exemple : Structure d'un programme

#include <stdio.h>

int main() {
    int a = 5;
    int b = 10;
    int somme = a + b;

    printf("La somme est : %d\n", somme);
    return 0;
}

Pourquoi apprendre le C ?

  • Comprendre comment fonctionne la mémoire (RAM).
  • Base syntaxique de nombreux langages (C++, Java, PHP, JS).
  • Performance d'exécution inégalée.

3. Langage PHP

PHP (Hypertext Preprocessor) est le langage roi du web dynamique côté serveur. Il permet de générer du HTML, traiter des formulaires et dialoguer avec des bases de données.

Exemple : Script de bienvenue

<?php
    $nom = "Alice";
    $heure = 14;

    if ($heure < 18) {
        echo "Bonjour " . $nom;
    } else {
        echo "Bonsoir " . $nom;
    }
?>

5. Méthode Merise

Avant de se lancer dans le code ou la création de tables SQL, il est impératif de réfléchir à la structure des données. Merise est une méthode d'analyse et de conception qui permet de modéliser un système d'information.

1. Le concept clé : Le MCD

Le Modèle Conceptuel de Données (MCD) est un schéma qui représente les objets de votre système et les liens entre eux.

  • L'Entité (Rectangle) : Un objet concret (Ex: Client, Produit).
  • L'Association (Ovale) : Un verbe qui lie les entités (Ex: Acheter).
  • Les Propriétés : Les informations de l'entité (Ex: Nom, Prix).

2. Les Cardinalités (Les règles du jeu)

C'est ce qui définit les quantités minimum et maximum dans une relation (0,1 - 1,1 - 0,n - 1,n).

AUTEUR 0,n --------- ( Écrire ) --------- 1,1 LIVRE
Lecture :
• Un AUTEUR peut écrire plusieurs (n) LIVRES.
• Un LIVRE est écrit par un seul (1) AUTEUR.

3. Du Schéma à la Base de Données

Une fois le dessin (MCD) terminé, on le traduit en tableaux logiques (MLD).

Règle de transformation :
Si j'ai une relation (1,n), je prends l'identifiant de l'entité forte et je le mets comme Clé Étrangère dans l'entité faible.

=> Table LIVRE (id_livre, titre, #id_auteur)

6. MySQL

MySQL est le SGBD relationnel le plus utilisé pour le web.

Les commandes essentielles (CRUD)

1. CREATE (Créer)

CREATE TABLE utilisateurs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(50),
    email VARCHAR(100)
);

2. SELECT (Lire)

SELECT * FROM utilisateurs WHERE id = 1;

3. INSERT (Ajouter)

INSERT INTO utilisateurs (nom, email) 
VALUES ('Jean', 'jean@gmail.com');
